# Деплой на REG.RU / ISPmanager

Проект является Node.js-приложением на Express + Socket.IO.

## Что заливать

Загрузить на сервер:

- `server.js`
- `package.json`
- `package-lock.json`
- `index.html`
- `style.css`
- папку `js/`
- папку `data/` можно создать пустой

Не загружать:

- `node_modules/`
- локальный `data/rooms.json`
- `.env`

## Настройки Node.js-приложения в ISPmanager

В разделе Node.js создать приложение:

- Версия Node.js: `18` или новее
- Режим: `production`
- Корневая папка приложения: папка проекта
- Файл запуска: `server.js`
- Команда установки зависимостей: `npm ci --omit=dev`
- Команда запуска: `npm start`
- Переменные окружения:
  - `NODE_ENV=production`
  - `PORT=<порт, который выдаст ISPmanager>`
  - `HOST=0.0.0.0`
  - `MONOPOLY_DATA_DIR=./data`

Если ISPmanager сам выдаёт порт приложению, используйте именно его в `PORT`.

## Домен и HTTPS

1. Привязать домен или поддомен к сайту.
2. Включить SSL-сертификат, например Let’s Encrypt.
3. Проверить открытие:
   - `https://ваш-домен/`
   - `https://ваш-домен/health`

`/health` должен вернуть JSON с `ok: true`.

## WebSocket / Socket.IO

Для онлайн-игры нужен WebSocket-прокси. В ISPmanager/Nginx должны проходить заголовки:

```nginx
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
proxy_set_header Host $host;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;
```

Если комнаты создаются, но второй игрок не получает обновления, почти всегда причина в WebSocket-прокси.

## Права на данные

Папка `data/` должна быть доступна на запись пользователю, от которого запущено Node.js-приложение.

Игра сохраняет комнаты в:

```text
data/rooms.json
```

Для стабильной работы не удаляйте этот файл во время запущенной игры.

## Обновление проекта

1. Остановить приложение в ISPmanager.
2. Загрузить новые файлы проекта.
3. Выполнить:

```bash
npm ci --omit=dev
npm run check
```

4. Запустить приложение.
5. Проверить `/health`.

## Рекомендации по стабильности

- Использовать Node.js 18+.
- Включить автозапуск приложения после перезагрузки сервера.
- Следить за свободным местом на диске.
- Периодически делать резервную копию `data/rooms.json`, если важны текущие комнаты.
- После правок всегда перезапускать Node.js-приложение и обновлять вкладки игроков.

## Быстрая проверка после запуска

1. Открыть сайт.
2. Создать комнату.
3. Открыть ссылку комнаты во второй вкладке или на другом устройстве.
4. Проверить чат.
5. Начать игру.
6. Проверить бросок кубиков и обновление состояния у обоих игроков.
